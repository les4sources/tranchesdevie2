# Les mangeurs

« Mangeurs », c'est le nom des **clients** dans l'admin. Chaque mangeur est
identifié par son **numéro de téléphone** (c'est sa clé unique). L'onglet
**Mangeurs** liste tout le monde.

![La liste des mangeurs](shot:customers-index)

## La fiche d'un mangeur

Clique sur un nom pour ouvrir sa fiche. Tu y trouves :

![La fiche d'un mangeur](shot:customer-show)

- **Ses coordonnées** — nom, téléphone, email.
- **Son portefeuille** — un solde prépayé (en euros) qu'il peut utiliser pour
  payer ses commandes. Voir plus bas.
- **Son historique** — toutes ses commandes passées.
- **Ses messages** — les SMS et emails qu'il a reçus (codes de connexion,
  confirmations, « commande prête »…).

## Le portefeuille

Le **portefeuille** est un solde d'avance. Le client le recharge, puis paie ses
commandes dessus — pratique pour les habitués et les commandes récurrentes du
calendrier.

Les mouvements sont typés : **recharge** (top-up), **débit de commande** et
**remboursement de commande**. Chaque opération est tracée, tu peux donc
toujours expliquer un solde.

## Envoyer un SMS

Depuis la fiche, l'action **Envoyer un SMS** permet d'écrire un message libre au
mangeur (par ex. le prévenir d'un imprévu). Les SMS automatiques (confirmation,
commande prête) partent tout seuls, tu n'as pas à les envoyer.

## Créer / modifier un mangeur

Bouton **Nouveau mangeur** pour en ajouter un. Tu peux aussi **Modifier** une
fiche pour corriger un nom ou un email. Le téléphone étant la clé du compte,
change-le avec prudence.
