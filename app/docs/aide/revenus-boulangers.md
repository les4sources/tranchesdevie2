# Les revenus boulangers

L'entrée **€** de la nav ouvre le calcul de **répartition des revenus** entre
les boulangers et Les 4 Sources. C'est le cœur comptable du projet : qui a
gagné quoi sur une période.

![Les revenus boulangers](shot:baker-revenue)

## Le principe

Pour une période donnée, l'écran calcule, à partir des commandes réellement
encaissées :

- le **revenu brut** généré ;
- la part **Les 4 Sources** (un pourcentage paramétrable, historisé) ;
- les **frais** communs (par ex. le transport) ;
- la **part de chaque boulanger**, selon les règles de partage en vigueur.

## Les partenariats de revenu

Certains boulangers **mettent leur revenu en commun** puis se le répartissent.
Ces regroupements (les **partenariats**) définissent qui partage avec qui et
dans quelle proportion. Ils se gèrent dans les **Paramètres → Partenariats**.

## Où sont réglés les taux ?

Les paramètres du calcul (taux Les 4 Sources, transport, parts individuelles)
sont **historisés** : quand on change un taux, l'ancien reste valable pour les
périodes passées. Tout se règle dans les [Paramètres](parametres). Cet écran-ci
ne fait que **lire et calculer** ; il ne change pas les règles.

> C'est l'écran à ouvrir en fin de période pour savoir combien reverser à
> chacun.
